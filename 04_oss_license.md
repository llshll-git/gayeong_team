# 4. 오픈소스 라이선스

## 4.1 대표 라이선스 비교

| 라이선스 | 특징 | 코드 수정 공개 의무 | 상업적 사용 |
|---------|------|---------------------|--------------|
| GPL | 강한 Copyleft | 필수 | 가능 |
| MIT | 매우 자유로움 | 없음 | 가능 |
| Apache 2.0 | 특허 보호 조항 포함 | 없음 | 가능 |

## 4.2 왜 라이선스가 중요한가
라이선스는 사용·배포·수정 과정에서 발생할 수 있는 법적 문제를 방지하고  
오픈소스 생태계의 신뢰성을 유지하는 역할을 한다.

## 4.3 잘못된 사용 사례
- 출처 표기 없이 재배포  
- GPL 코드를 활용하면서 소스 비공개  
- 라이선스 조건을 혼합하여 충돌 발생
