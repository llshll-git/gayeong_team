# 오픈소스소프트웨어 팀 보고서  
팀원: 조상혁, 승세훈, 김가영

---

## 📌 보고서 구성
보고서는 다음의 네 가지 주제로 이루어져 있습니다.

1. **OSS 개요** — 오픈소스의 의미와 특징  
2. **OSS 역사** — 오픈소스 소프트웨어의 탄생과 확산 과정  
3. **대표적 OSS 소개** — 실제 널리 활용되는 오픈소스 사례 분석  
4. **OSS 저작권** — 라이선스 종류와 특징  

모든 문서는 Markdown(.md) 기반으로 작성되며,  
각 문서는 팀원들의 Issue–PR 기반 협업 과정을 통해 발전했습니다.

---

## 🧩 GitHub 협업 요소 활용
- **Issue**: 각 주제별 작업 분배 및 논의  
- **Milestone**: 보고서 초안 → 검토 → 최종 제출 단계로 구분  
- **Label**: `content`, `review`, `fix`, `enhancement` 등 적용  
- **PR(Pull Request)**: 모든 문서는 팀원별 브랜치를 통해 작성 후 PR로 병합  
- **Wiki**: 정리된 핵심 개념과 참고 링크 정리  
- **Project 보드**: 칸반 방식으로 작업 흐름 관리(To do → In progress → Done)
